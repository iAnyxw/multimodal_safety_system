# 保研简历 — 项目经历

 **多模态安全审核系统——文本/图像/音频统一审核引擎**

**内容简介**：面向互联网内容安全审核场景，构建覆盖文本、图像、音频三种模态的统一安全审核系统。以 XLM-RoBERTa 为文本基座、CLIP 为图像基座、Whisper 为语音基座，设计多模态融合引擎与可配置策略引擎，实现端到端的风险判定与自然语言解释输出，并搭建 FastAPI + Streamlit 全栈应用。

**主要工作**：
- 从零构建 5,396 条中文安全审核数据集，覆盖辱骂、淫秽、身份仇恨三类多标签，正负比例 49:51，按有毒标签分层划分训练/验证集；
- 基于 XLM-RoBERTa-base 微调中文多标签文本分类模型，采用 FP16 混合精度训练与 linear warmup 策略，最终 Micro F1=0.954、Macro F1=0.952；通过 3-Fold 交叉验证（std=±0.003）排除过拟合，并对 0.3~0.7 区间进行阈值稳定性扫描，确认模型输出鲁棒；
- 设计 CLIP-ViT-Base-Patch32 多维度图像审核方案，构造 18 个分类 prompt（覆盖色情/暴力威胁/安全三类），采用分段阈值决策（>0.6 拦截、0.4~0.6 复审、<0.4 放行），并预留 Qwen2-VL-2B 深度审核接口用于疑图二次确认；
- 实现 Whisper-base → XLM-R 级联音频审核流水线，集成 noisereduce 降噪与 librosa 归一化预处理，复用文本模型实现端到端语音风险判定；
- 设计多模态融合引擎（FusionEngine），实现「一致强化（×1.2）/ 冲突降权（×0.8）/ 孤证降级」三种融合策略；设计策略引擎（PolicyEngine），支持 JSON 可配置规则将融合分数映射为 SAFE/REVIEW/UNSAFE 三级业务决策；
- 开发自然语言解释器（Explainer），自动将多模态审核结果转换为结构化中文描述（含风险评分、各模态明细、融合逻辑与处理建议）；
- 基于 FastAPI 构建 RESTful API（/predict/text、/predict/image、/predict/audio、/predict/multimodal），基于 Streamlit 实现可视化前端，以上前后端代码采用 Vibe Coding 方式完成；

 **多模态安全审核系统（精简版）**

**内容简介**：面向互联网内容安全场景，构建文本/图像/音频三模态统一审核引擎，以 XLM-RoBERTa、CLIP、Whisper 为各模态基座，设计融合引擎与策略引擎实现端到端风险判定与全栈部署。

**主要工作**：
- 自建 5,396 条中文安全审核数据集（3 类多标签），基于 XLM-RoBERTa 微调，Micro F1=0.954，3-Fold CV 验证无过拟合（std=±0.003）；
- 设计 CLIP 18 维度图像分类与 Whisper→XLM-R 级联音频流水线，覆盖图像与语音审核；
- 实现多模态融合引擎（一致强化/冲突降权/孤证降级）与可配置策略引擎，搭建 FastAPI + Streamlit 全栈应用（Vibe Coding）。
